### Phase 05 — Web Dashboard

User explicitly scoped these out of Plan 05-13 in `05-CONTEXT.md`; none block any roadmap success criterion:

- **WR-04** — `pipeline.ts:676 getDashboardHome` hardcodes `recent_versions: []`. Dashboard home API advertises the field but returns an empty list regardless of DB state.
- **WR-01** — `dashboard-routes.ts:227` uses hardcoded `'outputs'` path root; ignores configurable `outputRoot`. Breaks output-streaming route from a non-default CWD.
- **WR-05** — Dashboard `lib/api.ts fetchJson` discards error response body; typed server codes (`VERSION_NOT_FOUND`, `OUTPUT_UNAVAILABLE`, etc.) never surface in UI.
- **IN-01** — `qNum` query-param parser accepts negatives and non-integer floats; SQLite clamps silently but contract is undefined.
- **IN-02** — `: ping` keep-alive frame is `data: : ping\n\n` on the wire (not an SSE comment per the spec). Keeps TCP warm as intended; documentation-level inaccuracy.
- **IN-04** — `lib/shape.ts normalizeStatus` silently maps unknown status values to `'queued'`. Intentional "never unstyled" fallback; no longer load-bearing after CR-01 closure because the adapter emits union-valid statuses at source.
