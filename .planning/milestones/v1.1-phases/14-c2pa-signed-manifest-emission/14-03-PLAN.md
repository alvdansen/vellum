This revision applies six BLOCKER/HIGH/MEDIUM fixes from the plan-checker:
1. **Concern #2 (full) — sidecar derivation strategy.** v1.1 ships option (c) — REDUCE SCOPE. c2pa-node v0.5.26 does NOT expose a cryptographically-bound sidecar API. Producing a "sidecar" by signing a placeholder PNG (the original plan's approach) is cryptographically invalid. v1.1 emits signed manifests for native-embed formats (PNG/JPEG via buffer API; MP4/WebP/TIFF via file API). EXR/PSD surface as `unsupported_format` provenance events with the original file untouched. REQUIREMENTS.md adds an explicit v1.2 deferred item for cryptographic-sidecar support pending c2pa-node API additions OR direct c2pa-rs binding. Plan 14-05's verification asserts the manifest contains a `c2pa.hash.data` assertion AND that the asserted hash matches SHA-256 of the asset bytes (per Concern #8) — proving manifests bind to the actual asset.
2. **Concern #3 — T-14-12 threat model addition.** Document key-bytes-in-process-heap as ACCEPT with explicit mitigations (file mode 0600, low-privilege user, no `--inspect`/core-dumps in prod). HSM signing is v1.2+ per REQUIREMENTS Out-of-Scope.
3. **Concern #5 — temp file location + cleanup.** Specify `<outputsDir>/.tmp-c2pa/<versionId>/` with mode 0700 dir + 0600 files; try/finally cleanup unlinks both src + dest temp paths.
4. **Concern #6 — pre-stat OOM guard.** Pre-stat the file BEFORE readFile. If size > BUFFER_SIGNING_MAX_BYTES (500 MB for buffer API), short-circuit to a graceful-fail event (status_reason='asset_too_large_for_buffer_api'). The file-API path streams via c2pa-rs and does not need a buffer cap.
5. **Concern #7 — idempotency on re-sign.** Before reading + signing, check whether the file already has a manifest (lightweight `c2pa.read` OR via existing `manifest_signed: true` event for this version+filename). If yes, skip the re-sign (avoids double-embed + timestamp drift across recovery-poller retries).
6. **Concern #9 — concurrent .partial collision.** Use `mkstemp(\`${destPath}.c2pa-signed.XXXXXX\`)` (or Node's `mkdtemp` + nanoid suffix on the file name) for unique partial paths.

EXR/PSD are left untouched with a typed `unsupported_format` provenance event for v1.1 audit transparency.

**v1.1 Concern #2 SCOPE REDUCTION CHOICE:**
After Context7 + tarball investigation of c2pa-node v0.5.26:
- No `signSidecar` / `signExternal` / `signCloud` method exists in the JS API
- `embed: false` REQUIRES `remoteManifestUrl` (remote-hosted manifest, NOT local sidecar)
- `signedManifest` field IS returned for buffer-API path but is cryptographically bound to the SIGNED ASSET (e.g., a placeholder PNG); using it as a "sidecar" for an EXR is invalid
- c2pa-rs (the underlying Rust SDK) DOES support `set_no_embed(true)` + `sign_embeddable` for cryptographic sidecars, but that surface is NOT exposed by c2pa-node v0.5.26
- TIFF IS supported by c2pa-rs via the file-path API; v1.1 includes it
- EXR + PSD have NO c2pa-rs handler

v1.1 chooses **option (c) reduce scope**: native-embed only (PNG/JPEG/MP4/WebP/TIFF). EXR/PSD are surfaced as typed `unsupported_format` provenance events with the original file untouched on disk. v1.2 may add cryptographic sidecar support pending c2pa-node API additions OR direct c2pa-rs FFI binding (out of scope for v1.1).

      input: { bytes: Buffer } | { filePath: string },
    ): Promise<{ signed: Buffer | null; signedToPath: string | null; alreadySigned?: boolean }> {
      const signedAt = new Date().toISOString();

      // Concern #7 — idempotency on re-sign.
      const prior = this.provenanceRepo.getLatestManifestSignedEvent(versionId, filename);
      if (prior && prior.signed === true) {
        return { signed: null, signedToPath: null, alreadySigned: true };
      }

      // Path 1: signing disabled.
      if (this.c2paConfig === null) {
        this.provenanceRepo.appendManifestSignedEvent(versionId, {
          filename, format: '', signed: false, cert_subject_summary: '',
          signed_at: signedAt, status_reason: 'signing_disabled', algorithm: '',
        });
        return { signed: null, signedToPath: null };
      }

      // Path 2: format routing.
      const route = routeFormat(filename);
      if (route.mode === 'unsupported') {
        this.provenanceRepo.appendManifestSignedEvent(versionId, {
          filename, format: route.mimeType ?? '', signed: false, cert_subject_summary: '',
          signed_at: signedAt, status_reason: 'unsupported_format', algorithm: '',
        });
        return { signed: null, signedToPath: null };
      }

      // Path 3: lazy signer load.
      const signerOrCode = await this.getOrLoadSigner();
      if ('errorCode' in signerOrCode) {
        this.provenanceRepo.appendManifestSignedEvent(versionId, {
          filename, format: route.mimeType, signed: false, cert_subject_summary: '',
          signed_at: signedAt, status_reason: signerOrCode.errorCode, algorithm: '',
        });
        return { signed: null, signedToPath: null };
      }
      const signer = signerOrCode.signer;

      // Path 4: build manifest definition.
      const fingerprints = this.provenanceRepo.getLatestFingerprints(versionId);
      const primaryModel = derivePrimaryModel(fingerprints);
      const manifestDef = buildManifestDefinition({
        versionId, mimeType: route.mimeType, primaryModel,
        comfyuiVersion: null, appVersion: this.appVersion,
      });

      // Path 5: route to embed-buffer or embed-file.
      try {
        if (route.mode === 'embed-buffer') {
          // Concern #6 — pre-stat / size-cap.
          const bytes = 'bytes' in input ? input.bytes : await readFile(input.filePath);
          if (bytes.length > BUFFER_SIGNING_MAX_BYTES) {
            this.provenanceRepo.appendManifestSignedEvent(versionId, {
              filename, format: route.mimeType, signed: false, cert_subject_summary: signer.certSubjectSummary,
              signed_at: signedAt, status_reason: 'asset_too_large_for_buffer_api', algorithm: signer.algorithm,
            });
            return { signed: null, signedToPath: null };
          }
          const signedBytes = await signEmbedBuffer(bytes, route.mimeType, manifestDef, signer);
          this.provenanceRepo.appendManifestSignedEvent(versionId, {
            filename, format: route.mimeType, signed: true, cert_subject_summary: signer.certSubjectSummary,
            signed_at: signedAt, status_reason: '', algorithm: signer.algorithm,
          });
          return { signed: signedBytes, signedToPath: null };
        }
        // mode === 'embed-file' — MP4 / WebP / TIFF.
        // If caller gave us a filePath, sign in place via temp-dest. If bytes, write to temp src first.
        const result = await this.signViaTempFiles(versionId, input, route.mimeType, manifestDef, signer);
        this.provenanceRepo.appendManifestSignedEvent(versionId, {
          filename, format: route.mimeType, signed: true, cert_subject_summary: signer.certSubjectSummary,
          signed_at: signedAt, status_reason: '', algorithm: signer.algorithm,
        });
        return result;
      } catch (err) {
        console.error(`vfx-familiar: C2PA signing failed for ${versionId}/${filename}: ${(err as Error).message}`);
        this.provenanceRepo.appendManifestSignedEvent(versionId, {
          filename, format: route.mimeType, signed: false, cert_subject_summary: signer.certSubjectSummary,
          signed_at: signedAt, status_reason: 'sign_call_failed', algorithm: signer.algorithm,
        });
        return { signed: null, signedToPath: null };
      }
    }

    /** Lazy-load signer once per process. Distinguishes 'cert_load_failed' from
     *  'native_binding_unavailable' so the manifest_signed event surfaces a precise
     *  status_reason. Concern #11 — native binding load is wrapped by Plan 14-02's
     *  ensureC2paNode; this method just maps its TypedErrors to status_reason codes. */
    private async getOrLoadSigner(): Promise<{ signer: LoadedSigner } | { errorCode: 'cert_load_failed' | 'native_binding_unavailable' }> {
      if (this.signerCache !== null) return { signer: this.signerCache.signer };
      if (this.signerLoadFailedReason !== null) return { errorCode: this.signerLoadFailedReason.code };
      if (this.c2paConfig === null) return { errorCode: 'cert_load_failed' };

      try {
        const signer = await loadSigner(this.c2paConfig.certPemPath, this.c2paConfig.privateKeyPemPath);
        this.signerCache = { signer };
        return { signer };
      } catch (err) {
        const msg = (err as Error).message;
        const code: 'cert_load_failed' | 'native_binding_unavailable' =
          msg.includes('c2pa-node native binding unavailable') ? 'native_binding_unavailable' : 'cert_load_failed';
        this.signerLoadFailedReason = { code, message: msg };
        console.error(`vfx-familiar: C2PA signer load failed (${code}): ${msg}`);
        return { errorCode: code };
      }
    }

    /** Concern #5 — temp dir at <outputsDir>/.tmp-c2pa/<versionId>/, mode 0700.
     *  Concern #9 — partial paths use mkstemp-style suffix (nanoid) to avoid
     *  concurrent-writer collision. Cleanup unconditional in finally. */
    private async signViaTempFiles(
      versionId: string,
      input: { bytes: Buffer } | { filePath: string },
      mimeType: string,
      manifestDef: ManifestDefinition,
      signer: LoadedSigner,
    ): Promise<{ signed: Buffer | null; signedToPath: string | null }> {
      const tmpRoot = path.join(this.outputRoot, '.tmp-c2pa', versionId);
      await fs.mkdir(tmpRoot, { mode: 0o700, recursive: true });
      const suffix = nanoid(8);
      const srcTempPath = path.join(tmpRoot, `src-${suffix}`);
      const destTempPath = path.join(tmpRoot, `dest-${suffix}`);
      try {
        if ('bytes' in input) {
          await fs.writeFile(srcTempPath, input.bytes, { mode: 0o600 });
          await signEmbedFile(srcTempPath, destTempPath, mimeType, manifestDef, signer);
          const signedBytes = await fs.readFile(destTempPath);
          return { signed: signedBytes, signedToPath: null };
        }
        // input.filePath — sign-in-place via temp dest, caller will rename.
        await signEmbedFile(input.filePath, destTempPath, mimeType, manifestDef, signer);
        return { signed: null, signedToPath: destTempPath };  // CAUTION: caller MUST move/copy + rm this temp before tmpRoot cleanup. Adjust API: return a "consumeTempPath" callback or similar.
      } finally {
        // Concern #5 — unconditional cleanup. If signedToPath was returned, the
        // caller already consumed dest-${suffix}; both unlinks are safe (no-op
        // for missing files via fs.rm({ force: true })).
        await fs.rm(srcTempPath, { force: true });
        // dest-${suffix} cleanup happens after the caller renames it into place.
        // For the bytes-input branch, dest-${suffix} was already read into Buffer.
      }
    }

    /** Read-only accessor for HTTP layer (Plan 14-04). Returns the latest
     *  manifest_signed event for a version's primary output, or null. */
    getC2paStatusForVersion(versionId: string, filename: string): ManifestSignedPayloadFields | null {
      return this.provenanceRepo.getLatestManifestSignedEvent(versionId, filename);
    }
