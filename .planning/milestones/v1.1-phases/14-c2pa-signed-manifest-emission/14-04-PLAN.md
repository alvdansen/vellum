<objective>
Surface the signing layer at the HTTP + dashboard boundary. The output streaming route gains an X-C2PA-Signing-Status response header so clients can inspect signing state without parsing manifest bytes. The dashboard's VersionDrawer adds a C2PA signing badge.

**v1.1 scope reduction (Concern #2 from Plan 14-03 revision):** v1.1 does NOT ship a sidecar route or sidecar download link. c2pa-node v0.5.26 has no public sidecar API; producing pseudo-sidecars would be cryptographically invalid. EXR/PSD surface as `unsigned:unsupported_format` in the X-C2PA-Signing-Status header. v1.2 will add the sidecar route + dashboard link when:
- c2pa-node exposes signEmbeddable / sign_no_embed equivalent in its JS API, OR
- vfx-familiar binds directly to c2pa-rs (out of scope for v1.1)

The original Plan 14-04 included a sidecar route (Task 2) and dashboard link (Task 3); this revision REMOVES those tasks since they have no underlying mechanism in v1.1. The X-C2PA-Signing-Status header (revised Task 1) is the ONLY surfacing in v1.1.

**Architectural decision recorded (signing at write-time, not read-time):** The original D-CTX-8 considered signing at the HTTP layer at request time. Plan 14-03 shifted to engine-downloader hook (signing at write-time). Benefits preserved for v1.1: dual-transport parity for free since the file IS the source of truth; no signing latency on hot HTTP path; simpler crash safety.

**Dashboard format-table drift risk (LOW review note — accepted):** The dashboard was originally going to duplicate SIDECAR_EXTENSIONS for the sidecar-link conditional. With sidecar removal in v1.1, no format-table duplication is needed dashboard-side. The badge logic uses ONLY the engine-supplied status string, not extension matching. (See SUMMARY.md for the v1.2 plan when the sidecar route returns.)

Output: Updated output route + dashboard badge + tests covering signing-state response headers.
</objective>

       export type C2paStatus =
         | { status: 'signed' }
         | { status: 'unsigned'; reason: string }
         | { status: 'unknown' };

       /** Phase 14 — fetches X-C2PA-Signing-Status via HEAD and parses. Returns
        *  { status: 'unknown' } on network error or missing header (defence-in-depth). */
       export async function getC2paStatus(versionId: string): Promise<C2paStatus> {
         try {
           const res = await fetch(`/api/versions/${encodeURIComponent(versionId)}/output`, { method: 'HEAD' });
           const header = res.headers.get('X-C2PA-Signing-Status') ?? 'unknown';
           if (header === 'signed') return { status: 'signed' };
           if (header === 'unknown' || header === '') return { status: 'unknown' };
           if (header.startsWith('unsigned:')) return { status: 'unsigned', reason: header.slice('unsigned:'.length) };
           return { status: 'unknown' };
         } catch {
           return { status: 'unknown' };
         }
       }
