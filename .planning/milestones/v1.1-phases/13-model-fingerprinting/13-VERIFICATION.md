Three notable structural strengths worth flagging:

1. **Append-only via sibling event (T-13-07 mitigation):** The fingerprinter writes a NEW `models_fingerprinted` provenance event rather than updating `completed_event.models_json` in place. The architecture-purity invariant `grep this.db.update|this.db.delete` returns zero matches on `provenance-repo.ts`, and a regression test re-fetches the original completed-event row by id after append and asserts byte-equality on every field.

2. **Hot-path isolation by construction (criterion #4):** The Engine constructor binds a `void X.catch(...)` wrapped callback into GenerationEngine. The hook itself returns synchronously, and the async fingerprint work is scheduled to a microtask AFTER the generation hot path returns. `pipeline-fingerprint.test.ts` Test 4 asserts `getEventsForVersion(versionId).filter(e => e.event_type==='models_fingerprinted').length === 0` immediately after `getGenerationStatus(...)` returns 'completed' — the strictest behavioral signal that the hot path returns first.

3. **Idempotent crash recovery:** `Engine.fingerprintModelsForVersion` reads events first and returns early if a `models_fingerprinted` event already exists (line 715). A boot-time recovery sweep that calls the helper for every completed version is therefore O(N) reads + 0 hashes for already-done rows. Empty `models_json` arrays are also handled by writing an explicit empty fingerprinted event so idempotency holds even for the empty case.
