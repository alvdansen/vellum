<objective>
Plan 13-02 wires the Plan 13-01 fingerprinter into the completion path via an append-only sibling provenance event.

It does five things:
1. Extends `ProvenanceEventType` with `'models_fingerprinted'` and adds the matching payload to `ProvenanceEventPayload` in `src/store/provenance-repo.ts` (zero schema migration — the existing `event_type` text column has no DB CHECK constraint; only TS types gate the value).
2. Adds `appendModelsFingerprintedEvent(versionId, models)` (insert-only) and `getLatestFingerprints(versionId)` (read with fall-through to completed_event.models_json) to `ProvenanceRepo`.
3. Adds `Engine.fingerprintModelsForVersion(versionId): Promise<void>` to `src/engine/pipeline.ts` — reads latest models, hashes each via Plan 13-01's `fingerprintModel`, persists via the new repo method. Idempotent: re-runs are no-ops.
4. Hooks `void this.fingerprintHook(row.id)` (the Engine's bound callback) into `GenerationEngine.downloadAndPersist` immediately AFTER `writeCompletedEvent` returns — non-blocking (criterion #4). The hook is a constructor-injected callback so GenerationEngine does NOT take a back-reference to Engine (preserves layering).
5. Reads `process.env.VFX_FAMILIAR_MODELS_DIR` in `src/server.ts` and threads it into the Engine constructor options bag.

Purpose: closes ROADMAP success criteria #1 + #4 at the integration boundary (every model in the completed prompt blob ends up with a populated `model_hash` OR a typed `model_hash_unavailable` reason on the persisted sibling event, AND the generation hot path does not block on this work).

Output:
- ProvenanceRepo with two new methods + extended event-type union.
- Engine.fingerprintModelsForVersion async method.
- GenerationEngine constructor accepts an optional `fingerprintHook` callback; downloadAndPersist fires it via `void`.
- Engine ctor accepts `modelsDir` in its options bag and binds a hook that delegates to `this.fingerprintModelsForVersion`.
- src/server.ts threads `VFX_FAMILIAR_MODELS_DIR` env var.
- Two new test files: provenance-repo.test.ts cases for the new methods + pipeline-fingerprint.test.ts integration test for hot-path isolation + idempotency.

Wave 2 — depends on Plan 13-01 (consumes `MODEL_DIR_BY_CLASS`, `fingerprintModel`, and the extended `ModelRef.model_hash_unavailable` field).

       ```typescript
       getLatestFingerprints(versionId: string): ModelRef[] | null {
         const fingerprinted = this.db
           .select()
           .from(provenance)

           if (!Array.isArray(parsed)) return null;
           return parsed as ModelRef[];

           return null;
         }
       ```

       ```typescript
       // ================================================================
       // PHASE 13 — MODEL FINGERPRINTING (PROV-V-03)
       // ================================================================

       /**
        * Phase 13 — PROV-V-03. Fingerprint every loader-resolved model on the
        * completed prompt blob, persist as a sibling 'models_fingerprinted'
        * event (append-only — never UPDATEs the original 'completed' row).
        *
        * Idempotent (D-CTX `Claude's Discretion` clause): if a fingerprinted
        * event already exists for this version, returns immediately. Used by
        * crash-recovery on boot — re-running on an already-fingerprinted row
        * is a cheap no-op.
        *
        * Production / ComfyUI Cloud reality: when this.modelsDir is null,
        * every entry records `model_hash_unavailable: 'models_dir_not_configured'`
        * per D-CTX-5. Local-dev / self-host paths populate `model_hash` for
        * any model resolvable under the configured root.
        *
        * Background path: called from a `void`-fired callback at the
        * GenerationEngine.downloadAndPersist completion site. Errors are
        * logged via console.error in the caller's catch — never thrown.
        */
       async fingerprintModelsForVersion(versionId: string): Promise<void> {
         // Idempotency check — read events directly so we don't trigger the
         // fall-through-to-completed branch in getLatestFingerprints.
         const events = this.provenanceRepo.getEventsForVersion(versionId);
         const alreadyFingerprinted = events.some(
           (e) => e.event_type === 'models_fingerprinted',
         );
         if (alreadyFingerprinted) return;

         const source = this.provenanceRepo.getLatestFingerprints(versionId);
         if (!source) return; // pre-Phase-3 row or no completed event yet — skip
         if (source.length === 0) {
           // Empty models array — still record an explicit fingerprinted event
           // so the idempotency check above sees the work as done. Phase 14
           // can then rely on "fingerprinted event exists" as the signal that
           // the background path ran for this version.
           this.provenanceRepo.appendModelsFingerprintedEvent(versionId, []);
           return;
         }

         const fingerprinted: ModelRef[] = await Promise.all(
           source.map(async (m) => {
             const result = await fingerprintModel(
               this.modelsDir,
               m.class_type,
               m.model_name,
             );
             // Discriminated-union narrowing — exactly one of the two fields
             // is non-null per D-CTX-1.
             if ('model_hash' in result) {
               return {
                 node_id: m.node_id,
                 class_type: m.class_type,
                 model_name: m.model_name,
                 model_hash: result.model_hash,
                 model_hash_unavailable: null,
               };
             }
             return {
               node_id: m.node_id,
               class_type: m.class_type,
               model_name: m.model_name,
               model_hash: null,
               model_hash_unavailable: result.model_hash_unavailable,
             };
           }),
         );

         this.provenanceRepo.appendModelsFingerprintedEvent(versionId, fingerprinted);
       }
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Background fingerprinter → ProvenanceRepo writes | `fingerprintModelsForVersion` runs after the synchronous completion path returns. A crashed server during fingerprinting must not corrupt the completed event. Append-only sibling pattern means the original row is byte-stable; a missing fingerprinted event simply means "fingerprinter did not finish" — recoverable on next boot. |
