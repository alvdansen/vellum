<objective>
Surface the engine-layer divergence signal in the dashboard. When the user opens a reproduce-lineage version's drawer, auto-fetch the diff against its prior version, then conditionally render: (a) an amber WarningPill in the header when `reproduction_divergence !== null`, (b) a side-by-side parent-vs-reproduction `<img>` comparison block in the body when both outputs exist on disk.

Purpose: Close ROADMAP success criteria #1 and #2 at the dashboard boundary. Plan 12-01 established the field shape; this plan makes the divergence visible to the human VFX artist who would otherwise ship a "reproduction" they cannot see has drifted. Criterion #4 — bit-identical reproductions show no pill and no block — is enforced by leaning on the engine's `reproduction_divergence: null` semantics; the dashboard simply does not render the conditional UI when the field is null.

Output: A new `WarningPill` component, an extended `Version` type that carries `lineage_type`, an updated `VersionDrawer` with the auto-fetch effect + the two conditional renders, and component/integration tests covering all four divergence states. Dashboard build pipeline stays green.
</objective>

**2.2 — Add the auto-fetch effect:**

Inside `VersionDrawer` after the existing `useEffect` for provenance, add a second `useEffect` keyed on `[version.id, priorVersion?.id, version.lineage_type]`:

```typescript
// Phase 12 — DEMO-03 (D-CTX-2). Auto-fetch diff when this version is
// reproduce-lineage AND a priorVersion exists. Reuses the same `diff`
// state slot as the "View Diff" button so a subsequent click does not
// refetch.
useEffect(() => {
  if (version.lineage_type !== 'reproduce') return;
  if (!priorVersion) return;
  if (diff !== null) return; // already loaded — do not refetch
  let alive = true;
  diffVersion(priorVersion.id, version.id)
    .then((d) => {
      if (!alive) return;
      setDiff(d as DiffSummaryShape);
    })
    .catch(() => {
      // Network failure leaves diff=null; no pill, no block — graceful.
    });
  return () => {
    alive = false;
  };
}, [version.id, priorVersion?.id, version.lineage_type]);
```

Note: the dependency array includes `version.lineage_type` so a server-side change between renders (rare) is reflected. `diff` is intentionally NOT in the deps — including it would re-trigger the effect on every successful fetch.
