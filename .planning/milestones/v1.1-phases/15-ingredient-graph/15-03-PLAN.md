    /** Phase 15 — stream-hash a file's SHA-256 without loading bytes into memory. */
    async function streamSha256(filePath: string): Promise&lt;string | null&gt; {
      try {
        return await new Promise&lt;string&gt;((resolve, reject) =&gt; {
          const hash = createHash('sha256');
          const stream = createReadStream(filePath);
          stream.on('error', reject);
          stream.on('data', (chunk) =&gt; hash.update(chunk));
          stream.on('end', () =&gt; resolve(hash.digest('hex')));
        });
      } catch {
        return null;
      }
    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Engine.signOutput → manifest_signed event | The signed bytes are stable. The manifest_sha256 we record is an internal audit hash + parent-lookup hash — distinct from c2pa-node's per-ingredient labeled SHA. |
| Parent's manifest_sha256 + on-disk file → child's parentOf c2pa-node createIngredient | Reads from a row written earlier by the SAME repo path. Trust within process boundary. |
| Component image bytes (file) → c2pa-node createIngredient | Bytes from disk; we never execute. T-15-02 path-traversal guard preserved at the engine layer (matches Plan 15-01's hashComponentBytes guard). |
| Prompt blob → InputAssertion | T-15-01 mitigation: extractor is bounded + structured (Plan 15-01). Builder passes through verbatim (Plan 15-02). |
| Per-version signMutex → in-memory promise map | T-15-06 — bounded growth: settle cleanup is unconditional (try/finally). No leaks under recovery-poller storms. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Information Disclosure | Engine.signOutput → manifest payload | mitigate | extractInputAssertion (Plan 15-01) returns the structured shape; Engine.signOutput passes verbatim into buildManifestWithIngredients. NO workflow_json dump. |
| T-15-02 | Tampering / Path Traversal | Engine.buildManifestForVersion component file lookup | mitigate | The engine-side asset-ref resolution uses the same path-traversal guard as hashComponentBytes (rejects '..', '/', '\\', NUL); resolves via `nodepath.basename` defensively. |
| T-15-03 | Information Disclosure (stale) | Child manifest carries parent's stale manifest_sha256 after parent re-sign | accept (v1.1) | Re-sign idempotency from Plan 14-03 means the child only signs once. v1.2's version.export_manifest will re-derive on demand. |
