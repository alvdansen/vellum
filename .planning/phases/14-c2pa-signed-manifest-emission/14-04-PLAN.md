<objective>
Surface the signing layer at the HTTP + dashboard boundary. The output streaming route gains an X-C2PA-Signing-Status response header so clients can inspect signing state without parsing manifest bytes. A new sidecar route serves the .c2pa file for sidecar-mode formats. The dashboard's VersionDrawer adds a "Sidecar" download link conditional on format.

Purpose: Plan 14-03 signs at write-time; Plan 14-04 makes that signature visible to consumers. Per D-CTX-8 (download-time signing) the original CONTEXT plan considered signing at the HTTP layer — but Plan 14-03's design shifted signing to the engine's downloader hook, which is BETTER (signed file lives on disk; HTTP just streams it; dual-transport parity holds for free since the file IS the source of truth). This plan adapts: HTTP route streams already-signed bytes + reports status via header.

Output: Updated output route + new sidecar route + dashboard surface, all with tests covering signing-state response headers and sidecar visibility.
</objective>

    ```typescript
    // PHASE 14 — GET /api/versions/:id/output.c2pa
    //
    // Streams the C2PA sidecar manifest file for sidecar-mode formats (EXR, PSD,
    // TIFF). Returns 404 when format is embed-mode (manifest is in the asset
    // itself), or when the sidecar file is missing from disk.
    //
    // Path-traversal guard reused from the output route — same filename + same
    // basename normalization. T-14-03 mitigation.
    //
    // Caching: same policy as the output route (max-age=3600, immutable). The
    // sidecar bytes are content-addressed by versionId + filename and are
    // immutable once written.
    app.get('/api/versions/:id/output.c2pa', (c) => {
      const versionId = c.req.param('id');
      const version = engine.getVersion(versionId);  // throws VERSION_NOT_FOUND → 404

      // Determine signing state — must be sidecar-mode for this route to apply.
      const c2paStatus = engine.getC2paStatusForVersion(versionId);
      if (c2paStatus === null || !c2paStatus.signed || !c2paStatus.sidecar) {
        throw new TypedError(
          'SIDECAR_NOT_AVAILABLE',
          `No C2PA sidecar available for version '${versionId}' — format is embed-mode or signing was not performed`,
          'This format embeds the manifest in the asset itself. Use GET /api/versions/:id/output to retrieve the signed asset.',
        );
      }

      const raw = version.entity.outputs_json;
      let parsed: Array<{ filename?: string }> = [];
      if (raw) {
        try {
          const maybe = JSON.parse(raw);
          parsed = Array.isArray(maybe) ? (maybe as Array<{ filename?: string }>) : [];
        } catch {
          parsed = [];
        }
      }
      if (parsed.length === 0 || !parsed[0]?.filename) {
        throw new TypedError(
          'OUTPUT_UNAVAILABLE',
          `No outputs recorded for version '${versionId}'`,
          'The output file may not have been downloaded.',
        );
      }

      const storedFilename = parsed[0].filename;
      // T-14-03 reuse: same path-traversal rejection as the output route.
      if (
        storedFilename.includes('..') ||
        storedFilename.includes('/') ||
        storedFilename.includes('\\')
      ) {
        throw new TypedError(
          'INVALID_INPUT',
          `Invalid output filename '${storedFilename}' — contains path separator or traversal sequence`,
          'This is a bug or tampering attempt. Check the version record.',
        );
      }
      const filename = path.basename(storedFilename);
      const sidecarPath = path.resolve(engine.outputRoot, versionId, `${filename}.c2pa`);

      if (!existsSync(sidecarPath)) {
        throw new TypedError(
          'SIDECAR_NOT_AVAILABLE',
          `Sidecar file missing from disk for version '${versionId}'`,
          'The sidecar may not have been written yet. Check the X-C2PA-Signing-Status header on the asset endpoint for current signing state.',
        );
      }

      const nodeStream = createReadStream(sidecarPath);
      const webStream = Readable.toWeb(nodeStream) as ReadableStream<Uint8Array>;

      return c.body(webStream, 200, {
        // C2PA-registered MIME type per https://contentauthenticity.org docs.
        'Content-Type': 'application/c2pa',
        'Cache-Control': 'public, max-age=3600, immutable',
        'X-C2PA-Signing-Status': 'sidecar',  // self-describing
      });
    });
    ```
