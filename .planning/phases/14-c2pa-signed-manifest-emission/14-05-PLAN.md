c2pa.read() return shape (verified from c2pa-node tarball + types.d.ts):
```typescript
const result = await c2pa.read({ buffer, mimeType });  // null if no manifest, otherwise:
result.active_manifest: ResolvedManifest;
result.manifests: Record<string, ResolvedManifest>;
result.validation_status: ValidationStatus[];  // empty array = valid; codes per c2pa spec

// ResolvedManifest.assertions (from types.d.ts):
interface ManifestAssertion {
  label: string;       // 'c2pa.actions', 'c2pa.hash.data', 'c2pa.hash.bmff', 'c2pa.thumbnail.claim.jpeg', etc.
  data: any;           // assertion-specific payload
  kind?: 'Binary' | 'Cbor' | 'Json' | 'Uri' | null;
}

// c2pa.hash.data assertion data shape (per C2PA spec — Concern #8):
// {
//   exclusions: [{ start: int, length: int }],
//   alg: 'sha256' | 'sha384' | 'sha512',
//   hash: <bytes (base64 or array)>,
//   pad: <bytes>,
//   pad2?: <bytes>,
// }
//
// For BMFF (MP4): label is 'c2pa.hash.bmff' or 'c2pa.hash.bmff.v2'
// For PNG/JPEG: label is 'c2pa.hash.data'
```
