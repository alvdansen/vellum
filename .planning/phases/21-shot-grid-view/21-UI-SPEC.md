**Tab order through ShotGridView:**

2. TreeSidebar (existing tab order: workspace ŌåÆ project ŌåÆ sequence row ŌåÆ grid icon button)
3. Filter bar pills (left to right: All ŌåÆ wip ŌåÆ pending-review ŌåÆ approved ŌåÆ on-hold ŌåÆ omit*)

### Focus management

- **Page-load focus:** No autofocus. First Tab lands on the home button (head of document order).
- **View switch (home ŌåÆ shot-grid):** Focus moves to the sequence header chevron (the first interactive control inside the new view). Executor implements via a `useEffect` calling `ref.current?.focus()` when `activeView` changes to `'shot-grid'` AND the ref is attached.
- **View switch (shot-grid ŌåÆ home):** Focus moves to the home button itself (the affordance the user just activated).
- **VersionDrawer close:** Focus returns to the ShotGridCard button that opened it (Phase 5 drawer-close precedent already wired in `VersionDrawer.tsx`).
- **Filter pill click:** Focus stays on the clicked pill (default browser behavior ŌĆö do NOT shift focus on toggle).
- **"Show omitted" toggle:** Focus stays on the toggle. If toggling OFF reveals the omit pill is currently active, executor MUST reset `statusFilter = 'all'` AND update URL ŌĆö focus does not move.

### SSE-driven updates (D-22)

- Cards are keyed on `shotId` in the JSX (the `key={shot.id}` prop) so Preact reuses the DOM node and no `<img>` re-decode fires.

## URL State Contract (D-09)

The URL is the source of truth on first mount only; the signal is the source of truth thereafter. Updates flow via `history.replaceState` (not `pushState` ŌĆö view settings, not navigation events; mirrors Phase 18 precedent).

**URL shape:**

```
?seq={seqId}&view=shot-grid&statusFilter={all|wip|pending-review|approved|on-hold|omit}&showOmitted={0|1}
```

**Zod whitelist (executor implements):**

```typescript
const ShotGridUrlSchema = z.object({
  seq: z.string().min(1).optional(),
  view: z.enum(['home', 'shot-grid']).optional(),
  statusFilter: z.enum(['all', 'wip', 'pending-review', 'approved', 'on-hold', 'omit']).optional(),
  showOmitted: z.enum(['0', '1']).optional(),
});
```

**Malformed handling:** Catch the ZodError on `parse()`; do NOT throw to the caller. Fall back to defaults (`statusFilter='all'`, `showOmitted=false`, `view='home'`), log a `console.warn` with the malformed key, and proceed. Mirror Phase 18 `D-16` precedent.

**Precedence on first mount:** URL > signal > defaults. After first mount, signal is the only writer (URL mirrors via `replaceState`).
