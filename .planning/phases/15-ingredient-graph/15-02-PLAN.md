<objective>
Extend the Phase 14 manifest builder so it produces — IN ADDITION to the
Phase 14 ManifestDefinition — a list of IngredientSpec records that the
impure signer (Plan 15-03) feeds to c2pa-node's createIngredient +
addIngredient API. The pure builder ALSO emits two custom assertions in
ManifestDefinition.assertions: vfx_familiar.input (the structured prompt
+ sampler payload) and vfx_familiar.unavailable_ingredient (one entry per
spec whose bytes were unreachable at extraction time, so the audit trail
records what was attempted).

REVISION SCOPE (Path A — REWRITE per plan-checker B1):

The previous draft emitted parentOf / componentOf as c2pa.ingredient
entries inside assertions[]. That was wrong — c2pa-node's
BaseManifestDefinition deliberately EXCLUDES `ingredients` (and
`thumbnail`) from what `new ManifestBuilder(def)` accepts. Ingredients
are added AFTER construction via `manifestBuilder.addIngredient(storableIngredient)`,
where `storableIngredient` comes from `c2pa.createIngredient({asset, title, hash?})`.

The createIngredient API REQUIRES the asset bytes (BufferAsset or
FileAsset) — there is no public API to construct an ingredient from a
precomputed hash alone. So:

- Reachable ingredient bytes → drive c2pa.createIngredient → produce a
  StorableIngredient → manifestBuilder.addIngredient (this happens at
  the IMPURE signer layer in Plan 15-03)
- Unreachable ingredient bytes → record via vfx_familiar.unavailable_ingredient
  custom assertion (preserves the audit trail per ROADMAP criterion #5)

The pure builder's job is to produce: (a) the cleaned ManifestDefinition
(no ingredients field — c2pa-node's BaseManifestDefinition shape), (b) the
IngredientSpec recipe the impure layer drives, and (c) the
vfx_familiar.* custom assertions that DO land in assertions[].

Purpose: Plan 15-01 produced the data shapes. This plan produces the
contract the impure signer consumes. The signer module (Plan 15-03 also
modifies signer.ts) gets a NEW signWithIngredients entry point that
takes BuildManifestResult and threads ingredientSpecs through
createIngredient + addIngredient before sign.

Output: A BuildManifestResult that, when handed to the new signer entry
point with the asset bytes, produces a c2pa-signed asset whose
`Manifest.ingredients[]` (NOT assertions[]) carries parentOf +
componentOf, and whose assertions[] carries c2pa.created (Phase 14) +
vfx_familiar.input (Phase 15) + vfx_familiar.unavailable_ingredient[]
(Phase 15) for any spec the impure layer could not load bytes for.

## c2pa-node v0.5.26 — verified API used by this plan and Plan 15-03

The plan-checker spotted that the previous draft assumed ingredients
flow through assertions[]. The real path:

1. Pure layer: build BaseManifestDefinition (no ingredients field).
2. Impure layer: `const builder = new c2pa.ManifestBuilder(definition);`
3. Impure layer: for each spec with reachable bytes:
   ```ts
   const storable = await c2pa.createIngredient({
     asset: { buffer, mimeType } | { path, mimeType },  // BufferAsset OR FileAsset
     title: ingredient.title,
     hash: undefined,  // let c2pa-node compute its own labeled SHA (sha384 default)
   });
   storable.ingredient.relationship = Relationship.ParentOf | Relationship.ComponentOf;
   builder.addIngredient(storable);
   ```
4. Impure layer: `await c2pa.sign({ asset, manifest: builder, ... })`.

Key c2pa-node behaviors confirmed by source read:

- `createIngredient` REQUIRES the asset bytes — even if you supply `hash`,
  the binding still calls `bindings.create_ingredient(asset)` (bindings.js
  line 211) which streams the bytes through the native code. NO API
  exists to build an ingredient purely from a precomputed hash.
- `addIngredient` THROWS `IngredientHashMissingError` if ingredient.hash
  is not set after createIngredient returns. The default hash is the
  labeledSha output (sha384 base64 with format suffix, e.g.,
  `sha384-{base64}.png`). The labeledSha format is NOT a raw sha256 hex.
- `BufferAsset` signing is supported ONLY for `image/jpeg` and `image/png`
  (bindings.js line 132). For ingredient ASSETS we have more flexibility
  — c2pa-rs accepts buffer or file paths for ingredient extraction across
  more formats — but for the SIGNING ASSET, video / TIFF / WebP must use
  the file API.
- The default `ingredientHashAlgorithm` is `sha384` (hash.js line 18).
  c2pa-node lets you override via createC2pa({ ingredientHashAlgorithm }).

## Implication for Plan 15-03

The Engine.signOutput integration (Plan 15-03) becomes:
- For parentOf: read parent's signed file from disk (outputRoot/<parentId>/<parentFilename>) → pass as FileAsset to createIngredient → addIngredient with relationship=ParentOf. If file missing/unreadable → record via vfx_familiar.unavailable_ingredient.
- For componentOf: walk extractComponentIngredients results → for each, look up the input file at outputRoot/<versionId>/<filename> (Cloud-mode → typically not present per D-CTX-4) → if reachable, createIngredient + addIngredient with Relationship.ComponentOf. If unreachable → record via vfx_familiar.unavailable_ingredient.
- The component_unavailable taxonomy (file_not_found / file_unreadable) is preserved in the unavailable-assertion metadata, so audits can still see what was attempted.

     *         : { path: spec.assetRef.path, mimeType: spec.assetRef.mimeType },
     *       title: spec.title,
     *     });
     *     storable.ingredient.relationship = spec.relationship === 'parentOf'
     *       ? c2pa.Relationship.ParentOf
     *       : c2pa.Relationship.ComponentOf;
     *     builder.addIngredient(storable);
     *   }
     *   await c2pa.sign({ asset, manifest: builder, ... });
     *
     * Pure: zero I/O, synchronous. Idempotent (deeply-equal inputs →
     * deeply-equal outputs).
     */
    export function buildManifestWithIngredients(
      opts: BuildManifestWithIngredientsOptions,
    ): BuildManifestResult {
      const description = describePrimaryModel(opts.primaryModel);
      const assertions: ManifestAssertion[] = [
        {
          label: 'c2pa.actions',
          data: {
            actions: [
              {
                action: 'c2pa.created',
                digitalSourceType: IPTC_TRAINED_ALGORITHMIC_MEDIA,
                softwareAgent: { name: 'ComfyUI', version: opts.comfyuiVersion },
                parameters: { description },
              },
            ],
          },
        },
        {
          label: 'vfx_familiar.input',
          data: opts.ingredients.inputTo,
        },
      ];

      const ingredientSpecs: IngredientSpec[] = [];

      // parentOf — single spec when present.
      if (opts.ingredients.parentOf !== null) {
        const p = opts.ingredients.parentOf;
        const auditMetadata: Record&lt;string, string | number | null&gt; = {
          version_id: p.parent_version_id,
          lineage_type: p.lineage_type,
          manifest_hash: p.manifest_hash,
        };
        const assetRef = opts.ingredientAssetRefs.get('parent');
        if (assetRef &amp;&amp; assetRef.kind !== 'unavailable') {
          ingredientSpecs.push({
            relationship: 'parentOf',
            title: `Parent ${p.parent_version_id}`,
            assetRef,
            auditMetadata,
          });
        } else {
          // assetRef missing OR unavailable → record the dangling reference.
          const reason = assetRef?.kind === 'unavailable'
            ? assetRef.reason
            : (p.parent_unavailable ?? 'parent_manifest_pending');
          // Always also surface in ingredientSpecs (with assetRef='unavailable')
          // so Plan 15-03's signer can skip cleanly without lookup gymnastics.
          ingredientSpecs.push({
            relationship: 'parentOf',
            title: `Parent ${p.parent_version_id}`,
            assetRef: { kind: 'unavailable', reason },
            auditMetadata,
          });
          assertions.push({
            label: 'vfx_familiar.unavailable_ingredient',
            data: {
              relationship: 'parentOf',
              title: `Parent ${p.parent_version_id}`,
              reason,
              metadata: auditMetadata,
            },
          });
        }
      }

      // componentOf — one spec per ingredient. extractComponentIngredients
      // already returns the list sorted by node_id (Plan 15-01 contract).
      for (const c of opts.ingredients.componentOf) {
        const safeFilename = stripToBasename(c.input_filename);
        const auditMetadata: Record&lt;string, string | number | null&gt; = {
          node_id: c.node_id,
          role: c.role,
          input_filename: safeFilename, // T-15-04 — basename only
          class_type: c.class_type,
        };
        const title = `${c.role} image (${safeFilename})`;
        const assetRef = opts.ingredientAssetRefs.get(c.node_id);
        if (assetRef &amp;&amp; assetRef.kind !== 'unavailable') {
          ingredientSpecs.push({
            relationship: 'componentOf',
            title,
            assetRef,
            auditMetadata,
          });
        } else {
          const reason = assetRef?.kind === 'unavailable'
            ? assetRef.reason
            : 'file_not_found';
          ingredientSpecs.push({
            relationship: 'componentOf',
            title,
            assetRef: { kind: 'unavailable', reason },
            auditMetadata,
          });
          assertions.push({
            label: 'vfx_familiar.unavailable_ingredient',
            data: {
              relationship: 'componentOf',
              title,
              reason,
              metadata: auditMetadata,
            },
          });
        }
      }

      const definition: ManifestDefinition = {
        claim_generator: `vfx-familiar/${opts.appVersion} c2pa-node/${C2PA_NODE_VERSION}`,
        format: opts.mimeType,
        title: `Version ${opts.versionId}`,
        assertions,
      };

      return { definition, ingredientSpecs };
    }

    /** T-15-04 defence-in-depth: trust no caller, strip to basename. Pure
     *  string ops — no path module dependency to keep this module pure-string. */
    function stripToBasename(p: string): string {
      const lastSlash = p.lastIndexOf('/');
      const lastBackslash = p.lastIndexOf('\\');
      const idx = Math.max(lastSlash, lastBackslash);
      if (idx === -1) return p;
      return p.slice(idx + 1);
    }
