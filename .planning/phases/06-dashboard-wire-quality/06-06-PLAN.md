<objective>
SC-5: Hono's `streamSSE` helper exposes `writeSSE({ data: '...' })` which ALWAYS prepends `data: ` to each line. The current keep-alive at `sse.ts:207` calls `stream.writeSSE({ data: ': ping' })` which produces wire bytes `data: : ping\n\n` — that is NOT an SSE comment per the WHATWG spec (a comment line MUST begin with `:` as the FIRST character of the line). Browsers silently ignore the malformed frame, so the keep-alive intent works, but the wire shape lies. Fix by switching to the raw `stream.write(': ping\n\n')` path (`SSEStreamingApi extends StreamingApi`; the inherited `write(input: string)` writes raw bytes).

Purpose: Closes audit item IN-02. The wire shape now matches the spec — proxies and protocol analyzers see a real comment frame, browsers continue to ignore it (no behavioral change), and the in-line documentation accurately reflects what's on the wire.
