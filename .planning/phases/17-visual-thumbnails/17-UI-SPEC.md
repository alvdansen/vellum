### Phase 17 fixed-pixel exceptions (NOT on the 4px grid; locked by D-02 / D-13 industry conventions)

| Element | Width × Height | Rationale | Decision |
|---------|----------------|-----------|----------|
| **Server-encoded WebP** | up to 640 × 360 (capped longest edge; preserves source aspect per D-04) | Industry standard (Frame.io / Vimeo grid / NLE detail panels); retina-ready for 4K-monitor VFX review; ~25-40 KB per file | D-02 (LOCKED) |
| **VersionCard thumbnail render box** | `w-full` × auto via `aspect-video` (16:9 frame, height computed from width) | Existing VersionCard layout already uses `aspect-video` — no width change. Only `object-cover` → `object-contain` (D-19) | D-17 + D-19 (LOCKED) |
| **TreeSidebar shot-row thumbnail** | 80 × 45 (16:9, half of existing SkeletonThumbnail default) | TreeSidebar row density is tight (`px-2 py-1` per existing `TreeRow`); 160×90 is too large for a 280px-wide sidebar containing nested 12px-per-depth indentation. 80×45 fits comfortably with the shot label, satisfies retina at 1x, and keeps row height bounded under 56px including padding | **Researcher discretion** — addresses CONTEXT.md "Skeleton shimmer dimensions in TreeSidebar (existing default is 160×90; smaller may fit better in tree row density)" |
| **C2PA shield SVG overlay** | 20 × 20 in VersionCard thumbnails; 14 × 14 in TreeSidebar shot-row thumbnails | 20px = ~5% of 360px height — visible without dominating; matches Frame.io / Adobe overlay treatment. 14px scales proportionally for the 80×45 small variant (~31% height) | **Researcher discretion** — within CONTEXT.md "C2PA shield overlay treatment" sizing scope |
| **Shield offset from thumbnail bottom-right corner** | 6px right + 6px bottom (VersionCard); 4px right + 4px bottom (TreeSidebar small variant) | D-09 places shield at bottom-right; offset ensures visual breathing room without occluding the corner radius. 6px is on the implicit half-grid (4 + 2) acceptable for icon positioning per Phase 5 precedent | **Researcher discretion** — within D-09 |

### Thumbnail-specific color guarantees

1. **Letterbox bars MUST be transparent** (D-18). The server-side WebP is encoded at source aspect with alpha preserved; the CSS wrapper provides the bar color via its `background`. Light/dark theme switch is automatic; no theme-mismatch when the user toggles.

3. **Shield is rendered ONLY when `c2paStatus.status === 'signed'`** (D-10). Unsigned and unknown render NO overlay (no muted-color shield, no greyed-out state).

## Visual Hierarchy & Layout

Phase 17 thumbnails change the visual weight of two existing surfaces. This section declares the new hierarchy.

### Updated primary visual anchors (Phase 5 anchors carry forward; Phase 17 adjusts the version-card grid)

| View | Primary anchor (Phase 17) | Change from Phase 5 |
|------|---------------------------|---------------------|
| **Shot detail view** | The version-card thumbnail grid is now actually *visual* — each card shows the rendered output (not a generic placeholder). The grid remains the primary anchor; the thumbnails strengthen rather than replace this anchor. | UPGRADE: thumbnails were full-size `getOutputUrl()` images at 1:1 from disk (heavy, wasteful, slow). Now derived 640×360 WebP, lazy-loaded, fast. Anchor weight remains; per-card visual fidelity improves. |
| **TreeSidebar (left rail)** | Sidebar remains a SECONDARY persistent fixture. The new shot-row thumbnails (80×45) are quieter than the version cards (right pane) — same visual structure (16:9 frame, transparent letterbox), at half size, on the dominant canvas instead of the secondary surface. | UPGRADE: shot rows were text-only. Adding thumbnails enriches navigation without making the sidebar compete with the primary anchor. |
| **Version drawer** | Unchanged. C2PA TEXT pill (`<C2paBadge/>`) continues to surface signing state inside the drawer Output section. | NO CHANGE — D-11 locks: clicking the shield overlay bubbles to the parent VersionCard handler (opens the drawer); the drawer continues to show the full-text C2paBadge. |
| **Diff drawer** | Unchanged. | NO CHANGE — Phase 17 does not touch DiffDrawer. |

### Hierarchy enforcement rules (Phase 17 additions)

1. **Shield overlay never dominates the thumbnail.** Shield SVG is at most 6% of thumbnail height in the VersionCard context (20px / 360px after letterbox). Letterboxed source (e.g., a 1024×1024 square inside a 640×360 frame) means the shield sits on the LETTERBOX BAR, not on top of the source pixels — corrects for square-source crowding.
2. **Skeleton state semantics.** A skeleton in a VersionCard slot looks identical in shape to a rendered thumbnail (same `aspect-video` frame, same border-radius). The shimmer signals "data arriving" without changing layout (CLS=0 — REQUIREMENTS.md VIS-01 explicit).
3. **No visual decoration on the thumbnail itself.** No vignette, no inner shadow, no specular highlight, no rounded inner corners. The thumbnail is a faithful preview — auto-enhancement is an explicit anti-feature (REQUIREMENTS.md OOS).
4. **Shield does NOT have a separate visible focus ring.** Per D-11, the shield is non-interactive — no nested click target, no separate `tabindex`. The parent VersionCard's existing `focus-visible` ring suffices.

### `<Thumbnail/>` API contract (NEW component)

```typescript
export type ThumbnailSize = 'card' | 'sm';

export interface ThumbnailVersion {
  id: string;
  /** Output filename — needed for the URL since one version may have multiple outputs (research note: Phase 17 ships single-thumbnail-per-version; planner derives default filename) */
  filename?: string;
  /** Version status governs render path: 'complete' → real thumbnail; otherwise → skeleton */
  status: 'queued' | 'running' | 'complete' | 'failed';
  /** Free-form label used for accessible alt text */
  label: string;
}

export interface ThumbnailProps {
  /** Version metadata required for URL derivation + state-driven render */
  version: ThumbnailVersion;
  /** Size variant — 'card' = aspect-video full-width (VersionCard); 'sm' = 80×45 fixed (TreeSidebar) */
  size?: ThumbnailSize;
  /** C2PA status; if undefined or not 'signed', no shield is rendered */
  c2paStatus?: { status: 'signed' } | { status: 'unsigned'; reason: string } | { status: 'unknown' };
  /** Optional class to apply to the outermost wrapper (composition with parent) */
  class?: string;
  /** Optional ARIA label override; defaults to `Output for ${version.label}` (matches existing VersionCard string) */
  ariaLabel?: string;
}
```

**Render contract:**

| Condition | Render |
|-----------|--------|
| `version.status !== 'complete'` | `<SkeletonThumbnail />` at the size variant's dimensions, `aria-busy="true"` |
| `version.status === 'complete'` AND no thumbnail-fetch error | `<img loading="lazy" src={getThumbnailUrl(version.id)} class="object-contain" alt={ariaLabel ?? defaultAlt} />` wrapped in `<div class="relative aspect-video">`. If `c2paStatus.status === 'signed'`, append `<C2paShield/>` absolute-positioned bottom-right. |
| Image load error (browser `onerror` fires — server returned 404 / 500 / network) | Swap to `<SkeletonThumbnail />` AND set `aria-label="Preview unavailable for ${version.label}"`. Match the existing `SkeletonThumbnail` "no broken image icons" contract (D-07). |
| `aria-busy` lifecycle | Set to `"true"` while skeleton renders (loading or error), `"false"` once the real image has loaded (`onLoad` fires) |

**Dimensional contract:**

| `size` value | Wrapper class | Notes |
|--------------|---------------|-------|
| `'card'` (default) | `relative block aspect-video w-full overflow-hidden rounded` | Fills parent VersionCard's natural width; height is derived from 16:9 ratio |
| `'sm'` | `relative block aspect-video flex-shrink-0 overflow-hidden rounded` + `style={{ width: '80px' }}` | Fixed 80px width → 45px height via aspect-video; `flex-shrink-0` so the surrounding `TreeRow` flex container does not crush the thumbnail when the shot label is long |

**Lazy-load contract:**

- `<img loading="lazy">` is the SINGLE mechanism. NO IntersectionObserver shim (browser-native `loading="lazy"` is universally supported in 2026 evergreen browsers; the dashboard already requires desktop ≥ 1024px per Phase 5 D-WEBUI-17).
- `<img>` MUST declare explicit `width` and `height` attributes (not just CSS) to guarantee CLS=0 (REQUIREMENTS.md VIS-01). For `size='card'`, executor calculates from rendered viewport; for `size='sm'`, hardcode `width="80" height="45"`.
- For `size='card'` where width is fluid, executor MAY use `width="640" height="360"` as the intrinsic ratio attributes (browsers compute layout from the ratio, not the absolute pixel values, when CSS sets `width: 100%`).

**Click-target contract:**

- `<Thumbnail/>` itself has NO click handler (D-11). It is a presentational container.
- The parent `<VersionCard/>` already wraps everything in a `<button>` — the existing click handler bubbles. No change.
- The `TreeSidebar` shot row already wraps its content in a clickable `<div role="treeitem">`. The thumbnail sits inside that row; clicks bubble to the existing `onSelectShot` handler. No change.

### `<C2paShield/>` API contract (NEW component)

```typescript
export interface C2paShieldProps {
  /** Optional class for sizing (default-fallback: h-5 w-5 = 20×20) */
  class?: string;
  /** Optional title shown via native browser tooltip on hover. Defaults to "Signed · Verified provenance" per D-11 */
  title?: string;
}
```

**Render contract:**

```tsx
<svg
  class={class ?? 'h-5 w-5'}
  viewBox="0 0 24 24"             {/* 24×24 source viewBox — matches lucide convention */}
  role="img"
  aria-label={title ?? 'Signed · Verified provenance'}
  data-testid="c2pa-shield"
  style={{ filter: 'drop-shadow(0 0 1px rgba(0,0,0,0.6))' }}
>
  <title>{title ?? 'Signed · Verified provenance'}</title>
  {/* Adobe Content Credentials "CR" mark SVG path
      License: free for use to indicate Content Credentials per Adobe published guidance
      License-verification step: planner confirms terms before merging the SVG bytes
      Source: https://contentcredentials.org/icon */}
  <path d="..." fill="#FFFFFF" stroke="#1A1A1A" stroke-width="1.5" />
</svg>
```

**Decision rules:**

- **Inline SVG, NOT lucide-preact icon.** D-08 explicitly: "Adobe Content Credentials CR mark, NOT a generic shield-check." `lucide-preact` does not ship the official CR mark — using `<ShieldCheck/>` would violate D-08.
- **Static colors `#FFFFFF` body / `#1A1A1A` outline** (not theme-tokenized). The CR mark is a recognized brand element; tinting it would defeat its purpose as a regulator-recognized signal. Drop-shadow halo ensures legibility on bright thumbnails (white sky, snow, etc.).
- **NO interactive state.** No hover color change, no focus ring (the shield is non-interactive — D-11).
- **`role="img"` + `aria-label`** so screen readers announce "Signed, Verified provenance" when keyboard focus traverses the parent VersionCard.
- **`<title>` element inside `<svg>`** for the native browser tooltip (D-11). Setting `title=` on the `<svg>` element does NOT trigger a tooltip in all browsers — the inner `<title>` element is the spec-compliant approach.

### Phase 17 error states (no NEW typed-error codes — reuses Phase 5 + Phase 14 codes)

| Trigger | Code | UI behavior | Copy (toast — bottom-right) |
|---------|------|-------------|----------------------------|
| Server returns 404 from `GET /api/versions/:id/thumbnail` | (no code — HTTP 404) | `<Thumbnail/>` swaps to `<SkeletonThumbnail/>` + `aria-label="Preview unavailable for {label}"`. Silent — no toast. | (no toast — silent fallback) |

This matches D-07 ("no broken image icons, no distinct error skeleton") and the Phase 5 principle that toasts surface USER-actionable events.

## Accessibility Contract

Phase 17 adds new visual elements that MUST be accessible to keyboard and screen-reader users.

### Image-element accessibility

| Surface | Contract |
|---------|----------|
| `<img>` for completed version | `alt="Output for {version.label}"` — descriptive, references the user-meaningful version label (e.g., `Output for v003`). Empty `alt=""` is INCORRECT — these images are content, not decoration |
| `<img>` for skeleton state | NOT rendered — skeleton is a `<div role="presentation" aria-hidden="true">` (existing SkeletonThumbnail behavior). Add `aria-busy="true"` on the surrounding Thumbnail wrapper while loading |
| `<img>` after load failure | NOT rendered — falls back to skeleton with `aria-label="Preview unavailable for {label}"` set on the wrapper (informs SR users without a noisy error message) |

### C2PA shield accessibility

| Property | Value | Why |
|----------|-------|-----|
| `role` | `img` (on the `<svg>`) | Tells AT this is a meaningful image, not a decorative icon |
| `aria-label` | `Signed · Verified provenance` (D-11 verbatim) | Replaces the missing visible text label for SR users |
| `<title>` element inside `<svg>` | `Signed · Verified provenance` | Native browser tooltip on hover (D-11) — works without JS |
| Click target | NONE (D-11 LOCKED — clicks bubble to parent VersionCard) | Avoids the WCAG 2.5.5 "Target Size" issue of a 20px shield being too small to click |
| Keyboard focus | NONE on the shield itself | Same — no separate `tabindex`. Parent VersionCard's `<button>` already focusable |
| Tooltip on keyboard focus | The native `<title>` does NOT show on keyboard focus in most browsers. Mitigation: `aria-label` covers SR users. Sighted keyboard-only users discover signing state via the `<C2paBadge/>` text pill in the version drawer (which they reach by activating the focused VersionCard). | Acceptable per D-11 — full state is always available in the drawer |

### Lazy-load + CLS contract

| Property | Value | Verification |
|----------|-------|--------------|
| `loading="lazy"` on `<img>` | REQUIRED | REQUIREMENTS.md VIS-01 explicit |
| Explicit `width` and `height` HTML attributes | REQUIRED | REQUIREMENTS.md VIS-01 explicit ("CLS=0; explicit width/height") |
| `aspect-ratio` CSS hint (Tailwind `aspect-video`) | REQUIRED on wrapper | Reserves layout space before image decode |
| Reduced-motion `prefers-reduced-motion: reduce` | The 200ms fade-in is sub-threshold but executor MAY add a defensive media-query disable | Phase 5 inheritance |

### Color-contrast contract for the shield

The Adobe CR mark is a SYMBOL not text, so WCAG 1.4.3 (text contrast) does not apply directly. The mark's recognizability matters more than absolute contrast. Drop-shadow halo (`filter: drop-shadow(0 0 1px rgba(0,0,0,0.6))`) ensures the white shield body remains visible against bright source pixels. Verified visually against representative thumbnails (white sky, snow, blown highlights) at planning time — defer measurement to executor's visual QA pass.

## Responsive Contract

Phase 17 inherits Phase 5's responsive posture (≥ 1024px desktop required). Adds ZERO mobile breakpoints.

| Viewport | Phase 17 behavior |
|----------|-------------------|
| ≥ 1440px | TreeSidebar shot-row thumbnails at 80×45 fit comfortably alongside max-length shot names. VersionCard thumbnails fill the card width (typically 240-360px depending on grid columns) at native 16:9. |
| 1024-1439px | Same. TreeSidebar fixed at 280px; shot-row thumbnails preserve 80×45. VersionCard thumbnails scale fluidly to their container; aspect-video wrapper ensures CLS=0 across viewport changes. |
| < 1024px | Existing Phase 5 overlay shows: `Dashboard requires a desktop browser`. No thumbnail rendering needed below this breakpoint. |

**Adaptive density (TreeSidebar narrow widths):** D-decisions explicitly bound the TreeSidebar at 280px (Phase 5 D-WEBUI-17). Within that 280px, the shot-row layout is:

```
[ <indent: depth*12px> ][ <chevron 14px> ][ <thumbnail 80px> ][ <gap 4px> ][ <label flex-1 truncate> ]
```

Effective label area at depth 3 (shot): `280 - (8 + 3*12) - 14 - 80 - 4 = 138px`. Adequate for typical VFX shot names (`sh010`, `sh020_extended`, etc.). Long names truncate via existing `truncate` class on the label `<span>` — same behavior as today.
